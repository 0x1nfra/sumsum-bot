<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args -> paper runtime | Operator input controls fixture paths, interval, and database location. |
| signal/risk pipeline -> paper entry | Only accepted signals and persisted `ALLOWED` decisions may cross into position creation. |
| runtime loop -> filesystem storage | Long-running state must survive restarts without trusting in-memory caches. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | S | `paper_trader.py` command surface | mitigate | Keep a dedicated paper-only CLI and verify it does not import or expose any live credential or execution path. |
| T-04-02-02 | T | `core/paper_runtime.py` risk-to-entry seam | mitigate | Filter entries strictly to `RiskDecisionStatus.ALLOWED` and ignore blocked/rejected results before paper position creation. |
| T-04-02-03 | D | `core/paper_runtime.py` restart behavior | mitigate | Reload open positions from SQLite at the start of every pass so a process restart cannot orphan forward-test state in memory. |
</threat_model>
